Asynchrous server routine. Watches an input path to distribute work to clients.

Parameters:
  • path – input folder to watch for files.
  • outpath – output folder in which to place processed files.
  • output – output filename pattern. Reads are written in batches to file. If not given, output is to stdout.
  • port – port for server-client communication.
Returns:

None, and often won’t die gracefully.

pomoxis.apps.read_until_filter.read_until_align_filter(fast5, bwa_index, channels, start_port=5555, targets=['Ecoli', 'yeast'], whitelist=False)[source]¶

Demonstration read until application using scrappie and bwa to filter reads by identity.

Parameters:
  • fast5 – input bulk .fast5 file for read until simulation.
  • bwa_index – list of bwa index files (just basename without additional suffixes).
  • channels – list of channels to simulate.
  • start_port – port on which to run .fast5 replay server, bwa alignment server will be run on the following numbered port.
  • targets – list of reference names. If whitelist is False, reads aligning to these references will be ejected. If whitelist is True any read alinging to a reference other than those contained in this list will be ejected. Unidentified reads are never ejected.
  • whitelist – see target.
